Result grouping in Solr(1)

这一篇文章主要介绍solr中的result grouping基础功能，关于result grouping和facet、collapse & expand的区别我将会在接下来的文章中进行说明。

Result grouping 功能介绍

结果分组是将已经筛选过的结果通过指定的字段进行分组，字段值相同的文档会归入同一组中。

举个例子，如果你在淘宝搜索“苹果”，你可能会得到多个子类，分别是“手机品牌”、“电脑品牌”、“耳机品牌”，甚至是“水果”。每个子类又有多个结果，solr就会将包含这种层级结构的信息返回给用户。

Grouping 参数介绍

  • group

    • 设置为true表示结果分组功能启用。

  • group.field

    • 表示根据哪一个字段去进行分组。一次只能填入一个值，但可以设置多次。

  • rows

    • 返回的组数，默认值为10。

  • start

    • 表示在组列表里的初始偏移量。

  • sort

    • 整个groups应该如何排序。

  • group.limit

    • 每个group中返回的文档数，默认值为1。

  • group.offset

    • 表示每个组中文档列表里的初始偏移量。

  • group.sort

    • 在每个组里应该如何排序，不指明该值则按照sort排序。

  • group.format

    • “simple” 或者 “grouped”(默认值)。影响返回response的格式。

  • grouped.ngroups

    • 设置为true，会返回组的总数量。

Response

group.format会影响返回的response结构。
